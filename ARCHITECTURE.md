# Архитектура Telegram-бота для Одесской Епархии

## Обзор проекта
Telegram-бот для предоставления информации о священниках Одесской Епархии.

## Технологический стек
- **Язык**: Python 3.8+
- **Библиотека для Telegram**: python-telegram-bot
- **База данных**: SQLite
- **ORM**: SQLAlchemy (опционально) или прямой SQL

## Структура проекта
```
Cancellary_Bot/
├── bot.py                 # Главный файл бота
├── database.py            # Работа с базой данных
├── models.py              # Модели данных
├── handlers.py            # Обработчики команд
├── config.py              # Конфигурация
├── utils.py               # Вспомогательные функции
├── requirements.txt       # Зависимости
├── README.md             # Инструкции на русском
├── ARCHITECTURE.md       # Этот файл
└── database.db           # База данных SQLite (создается автоматически)
```

## Модель данных

### Таблица: priests (священники)
- `id` - уникальный идентификатор
- `name` - имя
- `surname` - фамилия
- `birth_date` - дата рождения
- `birth_place` - место рождения
- `status` - статус (Протоиерей, Иерей, Диакон, Протодиакон)
- `ordination_date` - дата рукоположения
- `service_place` - место служения
- `education` - место обучения
- `last_reward` - последняя награда
- `created_at` - дата создания записи
- `updated_at` - дата обновления записи

## Функциональность бота

### Команды:
1. `/start` - приветствие и инструкция
2. `/help` - справка по командам
3. `/search` - поиск священника по имени/фамилии
4. `/list` - список всех священников
5. `/status` - фильтр по статусу
6. `/add` - добавление нового священника (только для администраторов)
7. `/edit` - редактирование информации (только для администраторов)
8. `/delete` - удаление записи (только для администраторов)

### Интерактивные функции:
- Поиск с автодополнением
- Пагинация для больших списков
- Инлайн-кнопки для навигации
- Форматированный вывод информации

## Безопасность
- Проверка прав администратора для модификации данных
- Валидация входных данных
- Защита от SQL-инъекций
